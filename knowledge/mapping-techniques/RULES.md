# Rules — Mapping Techniques

Confirmed patterns to apply by default. Demote to `HYPOTHESES.md` if later contradicted.

---

**Build journey maps from research, never from internal assumption.** An assumed journey map is a communication artifact at best; it actively misleads teams at worst. Always ground stages and emotions in interview transcripts, session recordings, or support data before using the map to drive decisions.

**In story mapping, the backbone must describe user activities, not features.** If the top row reads like a feature list, the map has lost its diagnostic value. Rewrite each backbone item as a verb phrase from the user's perspective.

**In Wardley mapping, position components at their current maturity, not their target maturity.** Mapping aspirational positions hides the actual strategic gap and leads to incorrect make/buy decisions.
